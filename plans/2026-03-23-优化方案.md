## 优化点概述

### 1. 修复目标文件夹路径问题
**问题**: 当前影视资源被保存到网盘根目录下的"来自：分享"文件夹，而非配置的目标文件夹。

**原因分析**:
- `QuarkBrowser.save_share()` 方法中，文件夹选择逻辑存在问题
- 在文件夹选择弹窗中，可能没有正确选中目标文件夹就点击了确认
- 中文路径可能导致查找失败

**解决方案**:
- 修改 `quark/browser.py` 中的文件夹选择逻辑
- 使用逐级导航方式进入目标文件夹（如 filmTransfer -> movies）
- 路径改用英文：`filmTransfer/movies` 和 `filmTransfer/series`
- 新增 `_find_folder_by_name()` 方法，使用多种选择器策略查找文件夹

**修改文件**:
- `quark/browser.py` - `save_share()` 方法，改为传递完整路径并逐级导航
- `quark/client.py` - 传递完整路径给浏览器自动化
- `pipeline.py` - `_build_save_path()` 改用英文路径
- `main.py` - `history` 命令显示改用英文类型

**路径结构**:
- 电影：`filmTransfer/movies`
- 剧集：`filmTransfer/series`

**实施状态**: ✅ 已完成（2026-03-23 更新）

---

### 2. 同步历史记录与去重功能
**问题**: 每次执行任务时无法查看历史同步内容，可能导致重复保存。

**需求**:
- 记录每次同步的内容到文件
- 下次执行时检查历史记录，避免重复保存
- 支持按日期查看历史同步记录

**解决方案**:
- 新增 `SyncHistory` 类，管理同步历史记录
- 每次成功保存后，记录影视名称、ID、保存路径、时间戳到 JSON 文件
- 文件按日期命名：`sync_logs/sync_YYYY-MM-DD.json`
- 处理流程开始时，检查当日历史已保存的 ID，跳过已处理项

**修改/新增文件**:
- 新增 `storage/sync_history.py` - 同步历史管理模块
- 修改 `pipeline.py` - 在处理前检查历史记录
- 修改 `main.py` - 新增 `history` 命令查看历史记录

**数据格式**:
```json
{
  "date": "2026-03-23",
  "records": [
    {
      "timestamp": "2026-03-23T10:30:00",
      "item_id": "tt1234567",
      "item_name": "电影名称",
      "item_type": "movie",
      "addon_name": "Netflix Catalog",
      "resource_title": "资源标题",
      "resource_url": "https://pan.quark.cn/s/xxx",
      "save_path": "filmTransfer/电影",
      "status": "saved"
    }
  ]
}
```

**实施状态**: ✅ 已完成

### 3. 多备用转存路径与失效重试机制
**问题**: 当配置的转存目标文件夹不可用或分享链接失效时，没有备选方案。

**需求**:
- 配置多个备用转存路径
- 当主路径失效时，自动尝试备用路径
- 所有路径都失败时，记录到失败文件供人工排查

**解决方案**:
- 修改 `config.yaml`，将 `save_folder` 改为 `save_folders` 列表
- 按优先级顺序尝试每个路径
- 新增失败记录文件：`failed_transfers/failed_YYYY-MM-DD.json`

**配置修改**:
```yaml
quark:
  cookie: "..."
  # 转存路径列表（按优先级排序）
  save_folders:
    - "filmTransfer"           # 主路径
    - "filmTransfer_backup"    # 备用路径 1
    - "filmTransfer_alt"       # 备用路径 2
  auto_create_folder: true
```

**修改文件**:
- `config.yaml` - 改为列表配置
- `pipeline.py` - `_save_to_quark()` 方法增加多路径重试逻辑
- 新增 `failed_transfers/failed_YYYY-MM-DD.json` 记录失败项

**失败记录格式**:
```json
{
  "date": "2026-03-23",
  "failures": [
    {
      "timestamp": "2026-03-23T11:00:00",
      "item_id": "tt1234567",
      "item_name": "电影名称",
      "item_type": "movie",
      "addon_name": "Netflix Catalog",
      "attempted_paths": ["filmTransfer/电影", "filmTransfer_backup/电影"],
      "error_reason": "所有分享链接已失效",
      "tried_resources": [
        {"url": "https://...", "title": "..."},
        {"url": "https://...", "title": "..."}
      ]
    }
  ]
}
```

**实施状态**: ✅ 已完成

### 4. 文件日志系统（数据库接入前）
**问题**: 暂未接入数据库，所有记录需要保存到文件。

**需求**:
- 按日期记录每天不同的文件
- 为后期数据库接入做准备，保持接口一致性

**解决方案**:
- 统一日志目录结构：
  ```
  sync_logs/          # 同步历史记录
    sync_2026-03-23.json
    sync_2026-03-24.json
  failed_transfers/   # 失败记录
    failed_2026-03-23.json
    failed_2026-03-24.json
  ```
- `SyncHistory` 类提供统一的 CRUD 接口
- 后期可无缝切换到数据库版本

**修改/新增文件**:
- 新增 `storage/sync_history.py`

**实施状态**: ✅ 已完成

### 新增搜索源：b.funletu.com

**API 接口**: `https://b.funletu.com/search`

**请求方式**: POST (JSON)

**请求参数**:
```json
{
  "keyword": "关键词",
  "page": 1,
  "pageSize": 15,
  "categoryid": 0,
  "filetypeid": 0,
  "courseid": 1,
  "sortBy": "sort"
}
```

**响应格式**:
```json
{
  "code": 200,
  "msg": "请求成功",
  "data": {
    "list": [
      {
        "id": 3340439,
        "title": "<em>亲</em><em>爱</em><em>的</em><em>X</em>(2025)...",
        "url": "https://pan.quark.cn/s/5375a0c4de9c",
        "score": 22,
        "views": 24,
        "viewshistory": 24,
        "size": "共 6 个文件 4.5G",
        "uploaderid": "34*丽的菠萝的分享"
      }
    ],
    "total": 61
  }
}
```

**质量排序**:
- 优先使用 `score` 字段（资源质量评分）
- 次优先使用 `viewshistory` 或 `views` 字段（浏览历史/浏览量）
- 按 (score, views) 组合降序排序

**优势**:
- 直接返回夸克分享链接（`url` 字段），无需详情页跳转
- 自带质量评分系统，筛选更精准
- 响应速度快，加载稳定

**已弃用**:
- pan.52wei.cc（加载速度慢，已不再使用）
